# implement_20260420_210349.md

작성 일시: `2026-04-20 21:03:49 KST`

이 문서는 Parallel Agent Runtime v1과 Single-Agent Operator UX v1 구현 이후 남은 **미구현/후속 분리 항목**을 실제 구현 가능한 범위로 다시 고정하기 위한 개발 계획 문서다.

## 개발 목적
- v1에서 의도적으로 제외하거나 후속 후보로 남긴 항목을 `v1.5 hardening` 범위로 정리한다.
- 병렬 agent의 실사용 완성도를 높이되, 현재 안정화된 `722 pass / 0 fail / 125 files` 기준선을 깨지 않는다.
- write 병렬, 자동 검증, session export, owned-path guidance/warning를 순차적으로 보강한다.
- 구현 범위와 제외 범위를 명확히 나눠 v2 multi-agent swarm으로 과확장되지 않게 한다.

## 제외 범위
- tmux/iTerm2 swarm, teammate mailbox, TeamCreateTool류 다중 agent 채팅
- worktree 자동 merge, conflict 자동 해결, 자동 branch push
- child agent가 parent 권한보다 강한 권한을 획득하는 구조
- depth 3 이상 sub-agent 실행
- remote scheduler와 worktree 실행의 v1.5 통합
- `/agent resume` 실제 재개 구현

- LLM-as-Judge 기반 자동 완료 판정 교체
- 대규모 TUI redesign 또는 Ink dialog 기반 task manager
- tool-level owned path hard deny/transactional merge gate는 v2 후속으로 분리

## 공통 진행 규칙
- 각 Phase는 앞선 Phase의 자체 테스트 완료 후에만 시작한다.
- 구현 중 발생한 이슈는 해당 Phase에서 수정하고 기록한다.
- 체크박스 상태를 실제 진행 상태와 맞게 업데이트한다.
- 문서에 없는 범위 확장은 하지 않는다.
- write 병렬은 worktree 격리 또는 permission ask/deny 없이는 자동 실행하지 않는다.
- 자동 verifier는 기본적으로 advisory/background이며, user turn을 막는 blocking 동작은 v1.5에서 제외한다.
- session export는 summary/evidence 중심으로 제한하고 raw child transcript 전체 자동 포함은 제외한다.
- 전체 검증 기준은 `bun run typecheck`, `bun run build`, `bun test`다.
