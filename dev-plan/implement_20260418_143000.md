# implement_20260418_143000.md

작성 일시: `2026-04-18 14:30:00 KST`

이 문서는 coreline-agent에 clideck 연동 기반 5건(상태 신호, 프리셋/역할, 트랜스크립트, clideck preset, pipeline 통신)을 병렬로 도입하기 위한 작업 문서다.

## 개발 목적
- coreline-agent가 외부 도구(clideck 등)에 **관찰 가능한 상태 신호**를 노출한다.
- 역할/프리셋/프롬프트 라이브러리를 도입해 **반복 설정 비용**을 줄인다.
- 세션 기록을 정규화하고 검색/리플레이할 수 있게 해 **작업 이력 활용도**를 높인다.
- clideck에서 coreline-agent를 **managed agent preset**으로 등록/감시/제어할 수 있게 한다.
- 에이전트 간 **순차 결과 전달 체인(pipeline)**을 프록시 경유로 구현하여, 독립 프로세스 간 구조적 handoff를 가능하게 한다.

## 개발 범위

| WS | 워크스트림 | 소유 경로 |
|---|---|---|
| WS-A | 상태 관찰 신호 레이어 | `src/agent/status.*`, `src/proxy/server.ts`, `src/tui/status-bar.tsx` |
| WS-B | 프리셋/역할/프롬프트 라이브러리 | `src/config/roles.ts`, `src/prompt/library.ts`, `src/tui/slash-commands.ts` |
| WS-C | transcript 정규화 + 검색/리플레이 | `src/session/transcript.*`, `src/session/search.ts` |
| WS-D | clideck preset 지원 | `src/proxy/hooks.ts`, `docs/clideck-integration.md` |
| WS-E | Pipeline 통신 (프록시 경유 순차 handoff) | `src/agent/pipeline.*`, `src/agent/remote/pipeline-runtime.ts` |

## WS-E. Pipeline 통신 (프록시 경유 순차 handoff)

### 목표
- Agent-A의 결과가 Agent-B의 입력으로 자동 전달되는 **순차 handoff chain**을 구현한다.
- 기존 RemoteScheduler + proxy를 재사용하여, 독립 프로세스 간 구조적 통신을 가능하게 한다.

### 설계 개요

```
현재 (coordinator):
  Parent → [A, B, C] 병렬 → 결과 합산

추가 (pipeline):
  Parent → A → (A.result → B.context) → B → (B.result → C.context) → C → 최종 결과

프로세스 간 통신:
  터미널 1 (coreline-agent)
    └─ AgentTool pipeline: [stage-1, stage-2, stage-3]
         └─ stage-1 → POST proxy/v1/messages → proxy가 provider 선택 → 결과
         └─ stage-2 → POST proxy/v1/messages (body에 stage-1 결과 포함) → 결과
         └─ stage-3 → POST proxy/v1/messages (body에 stage-2 결과 포함) → 최종 결과
```

### Phase WS-E.1. Pipeline 타입 및 실행기
- [x] `src/agent/pipeline-types.ts` 신규:
  ```typescript
  interface PipelineStage {
    prompt: string;
    /** 앞 stage 결과를 이 stage의 context에 어떻게 주입할지 */
    contextPrefix?: string;  // 기본: "Previous stage result:\n"
    /** stage별 provider/model override (선택) */
    provider?: string;
    model?: string;
    /** stage별 timeout (선택) */
    timeoutMs?: number;
  }

  interface PipelineRequest {
    stages: PipelineStage[];
    /** 파이프라인 전체 목표 (각 stage system prompt에 주입) */
    goal?: string;
    /** 실패 시 정책: stop (기본) | skip | replan */
    onStageFailure?: "stop" | "skip";
  }

  interface PipelineStageResult {
    stageIndex: number;
    prompt: string;
    status: "completed" | "failed" | "skipped" | "aborted";
    text: string;
    usage: Usage;
    provider: string;
    model?: string;
    elapsedMs: number;
    error?: string;
  }

  interface PipelineResult {
    stages: PipelineStageResult[];
    finalText: string;
    totalUsage: Usage;
    completedCount: number;
    failedCount: number;
  }
  ```
- [x] `src/agent/pipeline-runner.ts` 신규:
  - `runPipeline(request: PipelineRequest, scheduler: RemoteScheduler | SubAgentRuntime, signal?: AbortSignal): Promise<PipelineResult>`
