# implement_20260415_205850.md

작성 일시: `2026-04-15 20:58:50 KST`

이 문서는 coreline-agent의 **3단계 품질 개선** 작업 범위를 고정하고, 이미 구현된 기능을 중복 구현하지 않으며, 진짜 에이전트로 가는 인프라를 준비하기 위한 작업 문서다.

## 개발 목적

1. **현재 어시스턴트 품질 다듬기** — 로컬 LLM에서 tool call 정확도 개선, 동일 tool 무한 반복 방지, TUI/CLI UX 폴리싱
2. **Memory 시스템 완성** — 이미 구현된 `src/memory/`, `src/tools/memory-read/`, `src/tools/memory-write/` 를 실제 동작 검증 + E2E 테스트 + 사용자 문서로 마감
3. **Plan-Execute-Evaluate 인프라 준비** — 클라우드 LLM 감지 시에만 활성화되는 타입/훅 레이어만 추가 (실제 Planner/Evaluator 구현은 다음 워크스트림)

## 개발 범위

### 우선순위 1: 어시스턴트 품질 다듬기
- Tool call 중복 감지기 (같은 tool + 같은 input 연속 3회 시 중단)
- 로컬 LLM 친화적 system prompt 변형 (tool schema JSON 간결화)
- ReAct fallback 파서 정확도 개선 (엣지 케이스 추가)
- TUI 개선: 토큰/턴 카운트 정확성, 상태바 라벨 명확화

### 우선순위 2: Memory 시스템 완성
- MemoryRead/MemoryWrite 실 E2E 시나리오 테스트 (Ollama + qwen)
- System prompt 주입 확인 (AGENT.md + MEMORY.md)
- 메모리 쓰기 후 재시작 시 로드 확인
- `docs/memory-system.md` 사용 예시 보강

### 우선순위 3: Plan-Execute-Evaluate 인프라
- Provider capability flag (`supportsPlanning: boolean` — 기본 false)
- 클라우드 LLM 자동 감지 (anthropic / openai / gemini → true)
- `PlanExecuteConfig` 타입 정의 (Planner/Evaluator 훅 인터페이스)
- 실제 Planner/Evaluator는 미구현 (next workstream)
- CLI flag `--plan-mode` 추가 (capability 있을 때만 동작)

## Phase 1. Tool call 중복 감지기

### 목표

로컬 LLM(Qwen 등)이 같은 tool을 같은 input으로 반복 호출하는 패턴을 감지하여 turn을 조기 종료하고, LLM에 "이미 시도했음" 피드백 전달. `--no-reasoning` 플래그와 함께 나타나는 무한 루프 현상 방지.

### 구현 태스크

- [x] `src/agent/tool-call-dedup.ts` 신규: `ToolCallDedup` 클래스
  - `record(toolName: string, inputHash: string): { isDuplicate: boolean; consecutiveCount: number }`
  - `hashInput(input: Record<string, unknown>): string` (canonical JSON + SHA256 8자)
  - `reset()` — 새 turn 시작 시 초기화
- [x] `src/agent/loop.ts` 통합:
  - 기본 임계치: 같은 `(toolName, inputHash)` 3회 연속 → `loop_detected` 이벤트 yield
  - 4회째부터는 tool 실행 대신 error result를 LLM에 전달 ("You've already tried this exact call 3 times — try a different approach")
  - `MAX_TOOL_CALL_DUPLICATES = 3` 상수
- [x] `src/agent/types.ts`: `LoopDetectedEvent` 타입 추가 (`AgentEvent` union에 포함)

### 자체 테스트

- [x] `tests/tool-call-dedup.test.ts` 신규

  - TC-1.2: 같은 tool + 다른 input → 각 고유한 hash, 반복 카운트 1
  - TC-1.3: 다른 tool 중간 삽입 → 연속 카운트 리셋
  - TC-1.4: `reset()` 호출 → 상태 초기화
