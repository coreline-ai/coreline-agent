이 문서는 coreline-agent가 외부 플러그인보다 먼저 갖춰야 할 **기본 내장 스킬 시스템**을 싱글 에이전트 성격에 맞게 도입하기 위한 작업 문서다.

## 개발 목적
- 사용자가 매번 `/dev-plan`, `/parallel-dev` 같은 지시를 하지 않아도, 에이전트가 작업 성격에 맞는 기본 작업 방식을 안전하게 선택하게 한다.
- 기본 스킬을 “도구 실행기”가 아니라 **작업 절차/전문 워크플로우 프롬프트**로 다뤄 기존 agentLoop, tools, permission, memory, plan-execute, reliability 구조를 깨지 않는다.
- 외부 사용자 설치형 플러그인 시스템보다 먼저, coreline-agent에 반드시 필요한 내장 전문성을 작고 보수적으로 탑재한다.
- prompt bloat, 자동 과발동, 권한 우회, sub-agent 전파, trace/privacy 누수를 막기 위해 deterministic local router와 명시 override를 제공한다.

## 공통 진행 규칙
- 내장 스킬은 **프롬프트 절차**다. Tool, MCP, AgentTool, PermissionEngine, HookEngine, Reliability layer를 대체하지 않는다.
- 자동 선택은 deterministic local rule 기반으로만 한다. 클라우드 LLM API 사용 금지.
- 기본 자동 선택은 root agent에만 적용한다. sub-agent는 부모가 명시 전달한 최소 guidance만 받는다.
- 자동 선택은 보수적으로 동작한다. 애매하면 선택하지 않는다.
- 자동 선택은 기본 1개 우선이며, 명시 선택 포함 한 턴 최대 3개, 총 본문은 기본 2,000자 이내로 제한한다.
- 사용자가 `--skill` 또는 `/skill use`로 명시한 스킬은 auto routing보다 우선한다.
- 사용자가 `--no-auto-skills` 또는 `/skill auto off`를 쓰면 자동 선택만 끈다. 명시 선택은 유지한다.
- 기본 동작 회귀가 없어야 한다. 스킬 미선택/비활성 시 기존 system prompt와 동일해야 한다.
- trace/export에는 기본적으로 `skill_id`, 선택 방식, timestamp만 남긴다. raw prompt와 selection reason 원문은 저장하지 않는다.
- 전체 검증 기준은 `bun run typecheck`, `bun run build`, `bun test`다. 현재 기준선은 `647 pass / 0 fail`이다.

## 핵심 설계 원칙

### 스킬의 정체성
```txt
Tool       = 실제 행동: FileRead, Bash, Git, MCP, Agent
Skill      = 작업 방식: dev-plan, investigate, review, parallel-dev
Role       = persona/style preset
Memory     = 프로젝트/세션 지식
Permission = 실제 허용/차단 권한
Hook       = 실행 전후 차단/관찰 경계
Reliability = 복구/검증/완료 판단 경계
```

### 3대 안전 원칙
```txt
Advisory only
  - 스킬은 작업 방식을 안내할 뿐 tool 실행 권한을 주지 않는다.
  - 스킬은 permission deny를 override하지 않는다.
  - 스킬은 hook blocking을 무시하지 않는다.

Root only
  - auto skill selection은 root agent에서만 실행한다.
  - sub-agent에는 parent가 명시한 최소 guidance만 전달한다.
  - sub-agent 내부에서 router를 재실행하지 않는다.

No override
  - 사용자 자연어가 system-level skill instruction을 무력화하지 못한다.
  - 자동 선택 off는 CLI/TUI 명령으로만 처리한다.
  - 스킬은 AGENTS/CLAUDE/project rule보다 높은 권한을 갖지 않는다.
```

### 우선순위 매트릭스
```txt
1. PermissionEngine deny / ask / allow
2. HookEngine blocking / reliability guard
3. 명시적 사용자 선택: --skill, /skill use
4. auto-skills off 여부: --no-auto-skills, /skill auto off
5. 모드 제약: chat / one-shot / plan / goal / autopilot / sub-agent
6. deterministic SkillRouter 결과
7. role preset / project persona
8. memory / AGENTS.md / CLAUDE.md / project instructions
9. 기본 system prompt
```

### 라우터 입력 정책
라우터는 사용자의 전체 확장 prompt를 그대로 보지 않는다.

```txt
User input
   │
   ├─ parse CLI/TUI explicit skill controls
   │
   ├─ buildRoutingText(raw top-level user intent)
   │      ├─ remove code fences
   │      ├─ remove block quotes / quoted examples
   │      ├─ exclude tool result / transcript / replay text
   │      ├─ exclude @file expanded file body
   │      └─ preserve short explicit user instruction only
   │
   ├─ BuiltInSkillRouter.select(routingText, context)
   │      ├─ explicit skills
   │      ├─ auto disabled check
   │      ├─ deterministic trigger match
   │      └─ budget/priority pruning
   │
   ├─ prepareUserPrompt / @file expansion
   │
   ├─ buildSystemPrompt(..., activeSkills)
   │      └─ # Active Built-in Skills section
   │
   └─ agentLoop
          └─ existing tools / permissions / memory / reliability
```
