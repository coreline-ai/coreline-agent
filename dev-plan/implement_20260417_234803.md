## 개발 목적
이미 실기능이 된 `--plan-mode`를 한 단계 더 완성해, TUI/세션/출력/실패 처리까지 포함한 일관된 plan 실행 경험을 만든다.

## 개발 범위
- TUI에서의 plan-mode 실행 흐름
- step 진행 표시 및 결과 출력 정리
- ambiguous / failed step 처리 UX 정리
- plan result 세션 저장/복원 규칙 정리
- plan-mode 관련 테스트와 최소 문서 반영

## 제외 범위
- multi-agent planner
- self-replanning / recursive replanning
- planner 성능 최적화 연구
- full visual plan editor
- 장기 메모리 기반 계획 최적화

- 2차 작업의 목표는 “계획 자체를 더 똑똑하게 만드는 것”이 아니라 “실사용 UX를 닫는 것”이다.
