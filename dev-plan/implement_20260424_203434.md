# implement_20260424_203434.md

작성 일시: `2026-04-24 20:34:34 KST`

이 문서는 `coreline-cli` 분석 결과 중 `coreline-agent`에 적용 가능한 1~6번 개선 범위를 고정하고, 구현이 목적 밖으로 확장되지 않도록 하기 위한 작업 문서다.

## 개발 목적

`coreline-agent`에 `coreline-cli`의 검증된 안전성/운영성 패턴을 선별 적용한다. 목표는 전체 구조를 Claude Code 계열 모노리스로 바꾸는 것이 아니라, 현재 `coreline-agent`의 provider/tool/hook/permission 아키텍처를 유지하면서 다음 6개 개선을 단계적으로 반영하는 것이다.

1. PostTool hook 연결
2. FileRead blocked device path 안전성
3. FileEdit 안정성 유틸 적용
4. filesystem permission hardening
5. Codex auth/config reader 보강
6. Bash security/parser 단계적 강화

## 개발 범위

- `src/hooks/*`, `src/tools/orchestration.ts`에 PostTool dispatch를 추가한다.
- `src/tools/file-read/*`에 device path block 및 회귀 테스트를 추가한다.
- `src/tools/file-edit/*`에 no-op, 대용량, UTF-16LE, quote normalization 관련 안정성 유틸을 추가한다.
- `src/permissions/*`에 protected path hardening을 추가하고 write 도구 권한 흐름에 연결한다.
- `src/providers/codex-backend.ts` 또는 분리 유틸에 `CODEX_AUTH_PATH`, `CODEX_CONFIG_PATH`, `~/.codex/config.toml` reader를 추가한다.
- `src/permissions/classifier.ts`와 BashTool 주변에 `coreline-cli`의 destructive warning / parser / shell quote / sed validation / read-only validation 중 결합도가 낮은 부분부터 단계 적용한다.
- 각 단계마다 단위 테스트와 기존 회귀 테스트를 실행한다.

## 제외 범위

- `coreline-cli` 전체 구조, Anthropic SDK fetch interception, embedded proxy 전체 포팅은 하지 않는다.
- Ink 6 / React 19 기반 UI 컴포넌트 직접 포팅은 하지 않는다.
- 공개 command hook, hooks.yml persistence, 외부 command hook 실행 기능은 이번 범위에서 제외한다.
- plugin marketplace, MCP full client, PowerShell tool, LSP tool, web browser tool은 제외한다.
- 자동 git merge/push/rebase, `/agent resume` 실제 재개 구현은 제외한다.
- `dist/`, `node_modules/`, `.env`는 수정하지 않는다.

## 문서 기반 제약 사항

- TypeScript ESM import에는 `.js` 확장자를 포함한다.
- 공유 타입 변경 시 모든 소비자를 typecheck로 검증한다.
- Hook failures는 fail-open이어야 하며, explicit `blocking: true`만 tool execution을 막는다.
- PreTool 순서는 기존과 동일하게 유지한다: permission deny → user approval → PreTool hook → tool execution.
- PostTool은 이번 Phase에서 broad dispatch를 구현하되 command hook은 추가하지 않는다.
- `PermissionEngine`의 deny/ask/allow 우선순위를 유지한다.
- 새 기능은 기존 테스트 skip/delete 없이 회귀 테스트를 추가한다.
- Phase 완료 조건은 자체 테스트 통과 + `bun run typecheck` clean이다.
