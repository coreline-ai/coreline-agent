## 공통 진행 규칙
- Phase 0에서 `ContextCollectionRequest`, `PromptMacro`, `BenchmarkScenario` 타입을 먼저 고정한다.
- 자동 수집은 advisory only. 파일을 읽기 전에 기존 permission/read-only 정책을 우회하지 않으며, `@file`/`prepareUserPrompt` 흐름을 재사용한다.
- macro는 명시 실행만 허용한다. 자동 상시 실행 금지. 순차 실행은 새 runner를 만들지 않고 기존 `pipeline-runner.ts`를 재사용한다.
- benchmark는 mock provider / deterministic test만 허용한다.
- 전체 검증은 `bun run typecheck`, `bun run build`, `bun test` 기준. 현재 기준선은 `647 pass / 0 fail`이다.

## 전문가 리뷰 반영 사항
- prompt macro는 새 순차 실행기를 만들지 않고 기존 `pipeline-runner.ts`를 얇게 감싸는 방식으로 구현한다.
- context collector는 기존 `@file`/`prepareUserPrompt`와 중복 구현하지 않고 “후보 추천” 역할에 집중한다.
- benchmark는 외부 LLM 호출 없이 mock provider + verification pack + CompletionJudge 호환 결과만 다룬다.
- macro cycle, max-step, oversized/binary/secret-like 파일 제외를 필수 테스트로 추가한다.
