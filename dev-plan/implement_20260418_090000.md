### WS별 메모

#### WS-1. humanInputMode 프록시 지원
- 현재는 **return / forbid 정책으로 구현 완료**됐다.
- 관련 근거:
  - `src/proxy/server.ts`
  - `src/proxy/v2.ts`
  - `tests/proxy-human-input.test.ts`
  - `tests/proxy-v2-endpoints.test.ts`
- interactive hosted tool + `human_input_mode=return`이면 HTTP 409 `human_input_required`를 반환한다.
- `forbid` 또는 미설정은 기존 400 경로를 유지한다.

#### WS-2. TUI proxy 상태 표시
- `--proxy [url]` 옵션과 `proxyStatus` 상태바 표시가 구현됐다.
- 관련 근거:
  - `src/index.ts`
  - `src/tui/app.tsx`
  - `src/tui/repl.tsx`
  - `src/tui/status-bar.tsx`
  - `tests/tui-proxy-status.test.ts`
- 로컬 proxy를 직접 띄우거나, 외부 proxy URL의 `/health` 상태를 반영할 수 있다.

#### WS-3. depth > 1 재귀 child
- AgentTool은 현재 **depth 2까지 허용**한다.
- 관련 근거:
  - `src/agent/subagent-types.ts`
  - `src/agent/subagent-runtime.ts`
  - `src/tools/agent/agent-tool.ts`
  - `tests/subagent-depth2.test.ts`
- depth 2는 read-only + Agent 금지 + 보수적 maxTurns/timeout 정책이다.

#### WS-4. Memory 자동 요약
- 대화가 정상 종료되면 root 대화에 한해 `auto_summary` 메모리를 자동 저장하도록 구현됐다.
- 관련 근거:
  - `src/memory/auto-summary.ts`
  - `src/memory/index.ts`
  - `src/agent/loop.ts`
  - `src/index.ts`
  - `tests/memory-auto-summary.test.ts`
- `--no-auto-summary` 및 `CORELINE_NO_AUTO_SUMMARY=1`로 비활성화 가능하다.

#### WS-5. Re-planner
- planner / runner / evaluator에 더해 **실패 task tail 재계획(replan) 경로가 구현됐다.**
- 관련 근거:
  - `src/agent/plan-execute/replanner.ts`
  - `src/agent/plan-execute/runner.ts`
  - `src/agent/plan-execute/types.ts`
  - `tests/plan-execute-replan.test.ts`
- 현재 범위는 completed prefix를 유지하면서 failed tail을 다시 구성하는 방식이다.

#### WS-6. Remote agent / distributed scheduler
- `src/agent/remote/` 모듈 구현 완료 (2026-04-18).
- 관련 근거:
  - `src/agent/remote/types.ts` — `RemoteAgentEndpoint`, `RemoteTaskRequest/Result`, `RemoteSchedulerConfig`, `RetryPolicy`
  - `src/agent/remote/client.ts` — `sendRemoteTask()`, `checkEndpointHealth()`, Anthropic Messages API 형식 HTTP 클라이언트
  - `src/agent/remote/scheduler.ts` — `RemoteScheduler` (windowed parallelism, round-robin, retry with backoff, abort 전파, health tracking)
  - `src/agent/remote/runtime.ts` — `RemoteSubAgentRuntime` (SubAgentRuntime 인터페이스 구현, 로컬 AgentTool과 투명 호환)
  - `tests/remote-agent-client.test.ts` — 8 tests (성공/timeout/500에러/abort/health)
  - `tests/remote-agent-scheduler.test.ts` — 6 tests (windowed concurrency/partial failure/retry/abort/health refresh)
- remote agent 구현 반영 후 405 tests pass / typecheck clean 검증 완료.

#### WS-7. D1 수동 TUI QA
- 실제 TUI를 직접 기동해 아래를 수동 확인했다.
  - `bun run src/index.ts --proxy` → local proxy online 상태 표시
