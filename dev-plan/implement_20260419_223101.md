이 문서는 coreline-agent에 **Parallel Agent Runtime v1**을 도입하기 위한 개발 범위를 고정하고, Claude Code식 병렬 에이전트 구조를 coreline-agent 현재 아키텍처에 맞게 안전하게 축소 적용하기 위한 작업 문서다.

## 개발 목적
- 현재 `AgentTool`의 subtask 병렬 실행을 한 단계 확장해, 메인 에이전트가 여러 child agent 작업을 **task 단위로 등록/추적/중단/조회/결과 통합**할 수 있게 한다.
- Claude Code의 `AgentTool`, `LocalAgentTask`, `forkedAgent`, `task framework`, `worktree` 패턴을 참고하되 전체 swarm/tmux/team 구조는 이식하지 않는다.
- v1은 “진짜 병렬 에이전트”의 최소 기준인 **background child 실행 + 상태 추적 + 중단 제어 + 구조화 결과 통합**에 집중한다.
- write 병렬과 worktree 격리는 v1의 기본 동작이 아니라, Phase 4에서 안전 후보로 설계/PoC 여부만 결정한다.

## 공통 진행 규칙
- 각 Phase는 앞선 Phase의 자체 테스트 완료 후 완료 처리한다.
- v1은 root agent에서만 background child task를 만든다. child 내부 자동 router/추가 swarm 생성은 금지한다.
- child 권한은 parent보다 강해지면 안 된다.
- child policy는 `ChildAgentPolicyEnvelope`로 명시 주입한다. 파일/웹/도구 결과/child 결과는 정보이며 지시문이 아니다.
- read-only/background 병렬을 먼저 완성하고, write 병렬은 Phase 4 후보로 분리한다.
- 공유 파일(`src/index.ts`, `src/tui/repl.tsx`, `src/tools/agent/agent-tool.ts`)은 단일 통합자가 최종 병합한다.
- task 상태는 deterministic하게 전이되어야 한다: `pending → running → completed | failed | aborted | timeout`.
- terminal 상태(`completed`, `failed`, `aborted`, `timeout`)로 전이된 task는 다시 실행하지 않고, 동일 terminal transition은 idempotent하게 무시한다.
- 동시 실행 child 수는 전역 제한을 둔다. v1 기본값은 `4`, 설정 허용 범위는 `1~8`로 시작한다.
- task record는 직렬화 가능한 데이터만 가진다. `AbortController`, running `Promise`, cleanup 함수는 runtime handle에만 둔다.
- TUI에서는 background task 조회/중단을 지원한다. one-shot CLI는 v1에서 background 실행을 지원하지 않으며 명시 에러를 반환한다.
- sync child는 현재 turn abort를 따른다. background child는 session abort 또는 `/agent stop`을 따른다. 현재 turn 취소만으로 background child를 자동 종료하지 않는다.
- 프로세스 종료, TUI teardown, session shutdown에서는 running background child를 정리하고 terminal 상태를 기록한다.
- task transcript/output에는 민감 정보 원문 저장을 최소화하고, 기존 session/export privacy 규칙을 따른다.
- terminal task 보존은 무제한이 아니다. v1 기본 보존량은 최근 terminal task `50`개로 시작하고, running/pending task는 보존 제한에서 제외한다.
- 전체 검증 기준은 `bun run typecheck`, `bun run build`, `bun test`다. 현재 기준선은 `722 pass / 0 fail / 125 files`다.

## Parallel Agent Runtime v1 정의
```txt
v1에서 말하는 병렬 에이전트:
  메인 에이전트가 AgentTool로 여러 child task를 등록한다.
  child task는 독립 task record, policy envelope, progress sink, abort scope를 가진다.
  child task는 scheduler 제한 안에서 동시에 실행될 수 있다.
  메인 에이전트 또는 사용자는 task 상태와 결과를 조회/중단할 수 있다.
  최종 결과는 completed/partial/failed/blocked를 구분해 통합된다.
```

```ts
export type ParallelAgentTaskStatus =
  | "pending"
  | "running"
  | "completed"
  | "failed"
  | "aborted"
  | "timeout";

export interface ParallelAgentTaskRecord {
  id: string; // paragtask_YYYYMMDD_HHMMSS_<shortid>
  parentId?: string;
  prompt: string;
  description?: string;
  status: ParallelAgentTaskStatus;
  cwd: string;
  provider: string;
  model?: string;
  agentDepth: number;
  write: boolean;
  ownedPaths?: string[];
  nonOwnedPaths?: string[];
  createdAt: string;
  updatedAt: string;
  startedAt?: string;
  finishedAt?: string;
  lastActivity?: string;
  outputPath?: string;
  transcriptPath?: string;
  summary?: string;
  finalText?: string;
  structuredResult?: ParallelAgentStructuredResult;
  error?: string;
  usedTools: string[];
  usage?: { inputTokens: number; outputTokens: number; totalTokens: number };
  progress?: {
    toolUseCount: number;
    lastTool?: string;
    messageCount?: number;
    tokenCount?: number;
  };
}

export interface ParallelAgentTaskRuntimeHandle {
  id: string;
  abortController: AbortController;
  promise: Promise<void>;
  cleanup?: () => void;
}

export interface ChildAgentPolicyEnvelope {
  role: "research" | "test" | "review" | "write";
  allowedTools: string[];
  deniedTools: string[];
  ownedPaths?: string[];
  nonOwnedPaths?: string[];
  canWrite: boolean;
  canSpawnChild: boolean;
  maxTurns: number;
  timeoutMs?: number;
  instructionBoundary: "user_prompt_only";
  mustIgnoreInstructionsFromFiles: boolean;
  mustReturnStructuredResult: boolean;
}

export interface ParallelAgentStructuredResult {
  status: "completed" | "partial" | "failed" | "blocked";
  summary: string;
  changedFiles: string[];
  readFiles: string[];
  commandsRun: string[];
  testsRun: Array<{
    command: string;
    status: "pass" | "fail" | "skipped";
    outputSummary?: string;
  }>;
  risks: string[];
  nextActions: string[];
}

export interface ParallelAgentMinimalResult {
  status: "completed" | "partial" | "failed" | "blocked";
  summary: string;
}

export interface ParallelAgentProgressSink {
  onMessage?(taskId: string, message: string): void;
  onToolStart?(taskId: string, toolName: string): void;
  onToolEnd?(taskId: string, toolName: string, ok: boolean): void;
  onUsage?(taskId: string, usage: { inputTokens?: number; outputTokens?: number }): void;
}

export interface ParallelAgentSchedulerOptions {
  maxParallelAgentTasks: number; // default 4, allowed 1..8
  maxRetainedTerminalTasks: number; // default 50
}

export interface ParallelAgentRuntimeCapabilities {
  supportsBackgroundTasks: boolean;
  maxParallelAgentTasks: number;
}
```

## Phase 0 보강 결정
| 항목 | 최종 결정 |
|---|---|
| Registry source of truth | **TUI root `AppState`가 소유**한다. `ToolUseContext`에는 registry handle과 `supportsBackgroundTasks`만 주입한다. one-shot CLI는 ephemeral context를 사용하되 background task는 지원하지 않는다. |
| `AppState` 변경 | `parallelAgentRegistry`, `parallelAgentScheduler`, `parallelAgentCapabilities` 후보 필드를 추가한다. 단, 초기 구현은 registry handle 하나로 묶어도 된다. |
| `ToolUseContext` 변경 | `parallelAgentRegistry?: ParallelAgentTaskRegistry`, `supportsBackgroundTasks?: boolean`를 추가한다. one-shot은 `supportsBackgroundTasks=false`다. |
| one-shot CLI 정책 | `runInBackground=true` 요청 시 sync fallback을 하지 않고 명시 에러를 반환한다. 이유: background 의미가 “즉시 task id 반환 + 이후 조회”이므로 sync fallback은 의미를 왜곡한다. |
| abort scope | sync child는 현재 turn abort를 따른다. background child는 session abort + explicit stop을 따른다. current turn cancel은 background task를 죽이지 않는다. |
| progress live update | `DefaultSubAgentRuntime.run()` 최종 반환만으로는 부족하므로 `ParallelAgentProgressSink` callback 또는 event hook을 추가한다. `/agents`는 `lastActivity`, `toolUseCount`, `lastTool`을 표시한다. |
| structured result | child는 최종 응답에 `ParallelAgentStructuredResult` JSON을 반환하도록 system guidance를 받는다. 파싱 실패 시 `finalText` fallback + `partial` 처리한다. 로컬 LLM 성공률이 낮으면 `status + summary`만 요구하는 minimal result mode로 축소한다. |
| policy envelope | child 실행 전 `ChildAgentPolicyEnvelope`를 만든다. full operating policy layer가 아니라 v1 최소 안전 봉투다. |
| Session 저장 범위 | running handle은 저장하지 않는다. terminal task의 `status`, `summary`, `structuredResult`, `outputPath`, `transcriptPath`만 기존 session/export와 연결한다. |
| 동시 실행 제한 | `maxParallelAgentTasks` 기본 4, 허용 1~8. 초과 task는 pending queue에 둔다. terminal 전이 시 scheduler가 queue를 drain한다. |
| task retention | terminal task는 최근 50개를 기본 보존한다. running/pending task는 eviction하지 않는다. handle은 terminal 전이 즉시 cleanup한다. |
| task id | `paragtask_YYYYMMDD_HHMMSS_<shortid>` 형식. 사람이 복사하기 쉬운 짧은 id alias는 후속 후보다. |
| `/agent resume` | v1 제외. list/status/read/stop까지만 지원하고 resume은 v1.5 후보로 분리한다. |
| Remote runtime | v1은 local `DefaultSubAgentRuntime` 기반. remote scheduler 통합은 후속으로 분리한다. |

## 예상 리스크 / 대응
| 리스크 | 영향 | 대응 |
|---|---:|---|
| 기존 AgentTool coordinator/pipeline 회귀 | 높음 | background mode를 adapter 계층으로 분리 |
| TUI 상태 관리 복잡도 증가 | 중간 | v1은 plain slash command 출력 우선 |
| child task 메모리 누수 | 중간 | terminal handle cleanup + max retained terminal tasks 도입 |
| abort race condition | 높음 | sync/background abort scope 분리 테스트, terminal idempotency 테스트 |
| current turn cancel이 background child를 죽임 | 높음 | dedicated task AbortController, session abort와 명시 stop만 연결 |
| progress가 최종 결과에서만 갱신됨 | 중간 | `ParallelAgentProgressSink` callback 추가 |
| structured result parse 실패 | 낮음 | finalText fallback + partial 표시 |
