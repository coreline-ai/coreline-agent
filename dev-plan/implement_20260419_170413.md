## 전문가 리뷰 반영 사항
- WebSocket은 1차 범위에서 제외하고, 기존 proxy SSE helper 기반 `text/event-stream`으로 status stream을 구현한다.
- `StatusTracker`에는 이미 `onStatusChange()`가 있으므로 새 subscribe 시스템을 만들지 않는다.
- A2A task send는 실제 AgentTool/remote scheduler 실행으로 바로 연결하지 않고 `accepted|disabled|rejected` adapter 응답까지만 구현한다.
- dashboard는 read-only shell이며 proxy auth/CORS/requestId 정책을 그대로 따른다.
- clideck은 기존 `docs/clideck-integration.md`와 `/hook/coreline/*` 계약을 재사용하고, clideck repo 수정은 하지 않는다.
