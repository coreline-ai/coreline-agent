작성 일시: `2026-04-17 23:48:04 KST`

이 문서는 MCP 운영성/권한 정리 작업의 범위를 고정하고, 현재 연결 가능한 MCP 기능을 안전하고 이해 가능한 운영 기능으로 다듬기 위한 작업 문서다.

## 개발 목적
이미 연결 가능한 MCP bridge를 운영 관점에서 정리해, 어떤 서버가 연결됐는지/어떤 권한으로 동작하는지/실패 시 어떻게 보이는지를 명확하게 만든다.

## 개발 범위
- MCP tool permission 정책 정리
- server별 on/off 및 선택 규칙 정리
- 연결 실패/timeout/error 가시화
- TUI 또는 상태 표시 영역에 MCP 상태 노출
- MCP 문서/설정 예시 보강
- 관련 회귀 테스트 추가

## 제외 범위
- MCP marketplace
- remote MCP transport 확장
- plugin 배포 시스템
- 완전한 승인 워크플로 엔진 재설계
- hosted tools와 MCP의 완전 통합 재설계

- 첫 목표는 “더 많은 MCP 기능”이 아니라 “현재 기능의 운영 안전성”이다.

- 연결 실패는 조용히 삼키지 말고 최소한 debug 또는 상태 영역에 드러나게 한다.
- 체크박스 상태를 실제 진행 상태와 맞게 업데이트한다.
- 문서에 없는 범위 확장은 하지 않는다.
