# implement_20260417_231050.md

작성 일시: `2026-04-17 23:10:50 KST`

이 문서는 hosted tools 프록시 전달(web_search, code_execution 등) 작업의 범위를 고정하고, 구현이 목적 밖으로 확장되지 않도록 하기 위한 작업 문서다.

## 개발 목적
proxy를 통해 hosted tools 요청을 전달할 수 있게 해, 외부 클라이언트가 tool-capable provider를 더 자연스럽게 사용할 수 있게 한다.

## 개발 범위
- hosted tools request shape 정리
- proxy mapper/server에서 hosted tools payload passthrough
- provider 지원 여부 분기
- tool 관련 응답/에러 매핑
- 관련 테스트 추가

## 제외 범위
- 신규 hosted tool 자체 구현
- 로컬 sandbox/code execution 엔진 구현
- MCP 연결
- human input mode

## 공통 진행 규칙
- passthrough가 목적이며, tool 기능 자체 재구현은 하지 않는다.
- provider가 지원하지 않는 경우 명확한 에러를 반환한다.
- 체크박스 상태를 실제 진행 상태와 맞게 업데이트한다.
- 문서에 없는 범위 확장은 하지 않는다.
