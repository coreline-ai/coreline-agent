## 1. Context (배경)

### 1.1 Why
Wave 1-10 완료 후 coreline-agent는 견고한 메모리·자가개선 시스템을 갖췄으나, **출력 품질을 객관적으로 평가/개선할 도구가 부재**:
- `summariseEval`은 pass_rate/turns/duration 등 **양적 지표**만
- 디자인·UX·콘텐츠 **질적 평가** 없음
- 사용자 브랜드 정체성을 메모리로 영속화할 패턴 없음
- AI 출력의 "slop" (자주색 그라디언트, 장식 이모지, 진부한 패턴) 자동 검출 불가

디자인 레퍼런스은 이러한 격차를 메우는 **검증된 개념 + 패턴**을 제공한다 — 단, 라이선스 제약으로 **개념만 차용**해야 한다.

### 1.2 Current State
- coreline-agent: 1414 tests, Wave 10 완료
- `MemoryType = "user" | "feedback" | "project" | "reference"` (4 types)
- `summariseEval`: 양적 평가만 (passRate, accuracy, durationMs)
- `~/.claude/skills/`: 디자인 레퍼런스 미설치 (현재 impl-plan, readme-style 등 심볼링크만 존재)
- 디자인 레퍼런스: 22개 references, 8개 starters, 24개 showcases, 37 SFX, 6 BGM, 시각 디자인 전용

### 1.3 Target State
구현 완료 후:
- `~/.claude/skills/디자인 레퍼런스/` 설치되어 사용자가 외부 에이전트(Claude Code/Cursor)에서 활용 가능
- `MemoryType`에 `"brand-spec"` 추가, `/memory brand-spec init|view|edit` 명령 동작
- `/critique <path>` — 5차원 평가 (Philosophy/Visual Hierarchy/Craft/Functionality/Originality) + Keep/Fix/Quick Wins 출력
- `/slop-check <path>` 또는 PostTool hook으로 anti-AI-slop 5+ 신호 자동 감지
- README + 라이선스 고지 일관 적용

### 1.4 Scope Boundary

**In scope**:
- Rank 1 설치 가이드 + 스크립트 (사용자 환경, coreline 코드 0)
- Rank 2 brand-spec MemoryType + slash 명령
- Rank 3 slop detector (5+ 패턴) + PostTool hook
- Rank 4 critique engine (LLM 기반) + slash 명령
- 모든 신규 모듈 헤더에 "Inspired by 디자인 레퍼런스" 명시
- README.md 라이선스 고지 추가

**Out of scope**:
- 디자인 레퍼런스 코드/asset/텍스트 직접 복사 (라이선스 위반)
- Rank 5 (20 디자인 철학 fixture) — 보류, Phase 5 옵션
- 디자인 레퍼런스의 HTML/React 컴포넌트 (TUI 부적합)
- 음향 효과 통합 (Ink는 오디오 미지원)
