이 문서는 현재 `coreline-agent` proxy에서 명시적으로 남아 있는 미지원/제약 지점(`human_input_mode`, provider별 hosted tools 지원 차이, streaming 제약 등)을 운영 계약 중심으로 다시 정리하고 후속 범위를 명확히 하기 위한 작업 문서다.

## 개발 목적
- proxy의 명시적 미지원/제약 사항을 코드/문서/테스트에서 일치시키고 운영 관점에서 더 명확하게 만든다.
- “지원 안 함”과 “부분 지원”과 “향후 고려”를 혼동하지 않게 정리한다.
- proxy 관련 후속 범위를 v3/v4 후보로 분리 가능한 상태로 만든다.

## 개발 범위
- `human_input_mode` 미지원 계약 재검토 및 문서/테스트 정리
- provider별 hosted tools 지원 범위 재정리
- provider streaming 제약 및 에러 surface 정리
- capabilities / operations 문서와 실제 응답 shape 일치 여부 점검
- 필요 시 최소한의 오류 메시지/상태 노출 개선

## 제외 범위
- full human input mode 구현
- proxy 아키텍처 재설계
- provider 신규 추가
- MCP와 proxy의 통합 재설계
- multi-tenant auth / rate limiting 시스템 확장
