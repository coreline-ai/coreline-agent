- Track 3의 컨텍스트 압축/Rate Limit/자기 평가/적응형 프롬프트는 후속 확장으로 분리 가능한 최소 계약만 고정한다.

## 공통 진행 규칙
- Phase 0 계약 고정 후 WS-1~WS-4는 병렬 착수 가능하다.
- `src/index.ts`, `src/tui/repl.tsx`, `src/tui/slash-commands.ts`는 최종 통합자가 한 번에 연결한다.
- Git 도구는 **단일 `Git` tool + `action` 필드**로 고정한다. `status|diff|log|show`는 read-only, `apply|stage|commit`은 permission ask를 유지한다.
- 캐시는 명시적 TTL/무효화 규칙이 있어야 하며, stale read가 파일 수정에 영향을 주면 안 된다. 공통 계약은 `ToolCache.invalidatePath(path)` / `invalidateAll()`로 고정한다.
- 트랜잭션은 기존 `BackupStore`와 충돌하지 않고 그 위에 얹는다.
- 전체 검증은 `bun run typecheck`, `bun run build`, `bun test`를 기준으로 한다. 현재 기준선은 `647 pass / 0 fail`이다.

## 전문가 리뷰 반영 사항
- Git 도구는 다중 도구가 아니라 **단일 `Git` tool + action enum**으로 구현한다. 도구 수를 줄여 provider tool schema와 permission 정책을 단순화한다.
- `ToolCache`는 FileRead/Glob만의 내부 최적화가 아니라 FileWrite/FileEdit/transaction rollback이 호출할 수 있는 공통 invalidation 계약을 가진다.
- Track 3은 이 문서에서 실제 구현하지 않고 타입/옵션 이름과 source-of-truth 충돌 여부만 기록한다. 실제 구현은 별도 dev-plan로 분리한다.
- silent failure 방지를 위해 non-git repo, huge diff, symlink/realpath, rollback partial failure를 필수 테스트에 포함한다.
