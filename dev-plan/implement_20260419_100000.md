## 085555 (신뢰도 레이어)와의 실행 순서

이 문서(hardening)와 `implement_20260419_085555.md`(신뢰도 레이어)는 서로 다른 레이어를 다루지만 `src/agent/loop.ts`, `src/index.ts`에서 충돌할 수 있다.

**고정 실행 순서:**
```
085555 Phase 0 (source-of-truth 계약)  ← 반드시 선행
    ↓
085555 Phase 1~4 (Judge/Trace/Recovery/VerificationPack)
    ↓
이 문서 Phase 0 + WS-1~4 (백업/Diff/테스트/비용)
    ↓
085555 Phase 5 (순차 통합, loop.ts 최종 연결)
```

**연결 규칙:**
- 이 문서의 WS-3 `TestRunResult`는 085555의 `VerificationPack` evidence 소스가 된다. 하지만 이 문서에서는 `TestRunResult`만 구현하고, evidence 연결은 085555 Phase 5에서 한다.
- 이 문서의 WS-4 `CostTracker`는 `loop.ts`에 `addUsage()` 호출을 추가하지만, 085555 Phase 5의 `loop.ts` 통합보다 먼저 병합한다.
- 085555 Phase 0의 source-of-truth 계약이 확정되기 전에는 이 문서의 타입이 085555 타입을 참조하거나 대체하지 않는다.

## 공통 진행 규칙
- 4개 WS는 Phase 0 이후 병렬 착수 가능 (WS-2만 WS-1 이후).
- 현재 테스트 기준선: `509 pass / 0 fail`. 구현 후 회귀 없이 유지.
- 각 Phase는 앞선 Phase의 자체 테스트 완료 후에만 완료 처리.
- 문서에 없는 범위 확장은 하지 않는다.
