이 문서는 `@file` 기능 전용 코어 구현 범위를 고정하고, 런타임 wiring 전 단계의 작업이 목적 밖으로 확장되지 않도록 하기 위한 작업 문서다.

## 개발 목적
- `@path` 토큰을 파싱하고 파일 내용을 첨부할 수 있는 코어 모듈을 만든다.
- 런타임/CLI/TUI 통합은 제외하고, 다음 워커가 붙기 쉬운 단순한 API를 확보한다.
- 중복 제거, 파일 크기 제한, binary/누락 오류 처리까지 코어 수준에서 정리한다.

## 개발 범위
- `@file` / `@path` 토큰 파싱
- 단일 파일 및 다중 파일 첨부 확장
- glob 패턴 확장 지원 여부 검토 및 코어 구현
- 중복 제거
- 파일 크기 제한
- binary / 누락 / 디렉토리 / glob 미스매치 오류 처리
- 전용 테스트 추가

## 제외 범위
- `src/index.ts` wiring
- `src/tui/*` 입력 UI 연동
- `src/agent/*` system prompt / context 연동
- `src/config/*` 설정 저장/재로딩 변경
- 실제 모델 요청 payload 통합
- permission engine 또는 session storage 변경
