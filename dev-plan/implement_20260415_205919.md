## 개발 목적
`coreline-agent`에 Claude Code 스타일의 서브 에이전트 위임 기능을 MVP 수준으로 도입한다. 첫 버전은 “깊이 1, read-only 중심, 결과 요약 반환”에 집중해 안전성과 통합 난이도를 낮춘다.

## 개발 범위
- `AgentTool` MVP 추가
- 부모 agent loop 내부에서 독립 child agent loop를 실행하는 `SubAgentRuntime` 추가
- child tool allowlist, depth limit, abort 전파, permission 하향 상속 적용
- parent 컨텍스트에는 child 전체 transcript 대신 요약 결과만 반환
- CLI/TUI 공용 런타임 경로에서 동일하게 동작하도록 wiring
- unit/integration 테스트 추가

## 제외 범위
- child agent의 write 계열 도구 사용 (`FileWrite`, `FileEdit`, `MemoryWrite`)
- child agent가 또 다른 child agent를 생성하는 재귀 실행 (depth > 1)
- child transcript 전체를 TUI에 실시간 스트리밍하는 UI
- child session 저장/`--resume` 복원
- provider별 별도 child model 라우팅, multi-provider fan-out
- swarm coordinator, task scheduler, remote agent

## MVP 아키텍처 초안

### 핵심 판단
- v1은 **진짜 child agent loop**를 사용한다.
- 단, **동시 다중 child 실행**은 하지 않는다.
- child는 **같은 provider 인스턴스**를 재사용하고, `AgentTool.isConcurrencySafe() = false`로 고정해 provider 동시성 복잡도를 피한다.
- child는 **read-only allowlist**만 사용하고, permission `ask`가 필요한 동작은 내부적으로 허용하지 않는다.
- `Agent` tool 기본 permission 정책은 **allow**로 두고, 필요 시 `permissions.yml`에서 상향 제한한다.

### 런타임 흐름
```text
Parent agentLoop
  └─ tool_use: Agent
      └─ AgentTool.call()
          └─ SubAgentRuntime.run(request)
              ├─ child tools 필터링 (read-only allowlist ∩ 요청 subset)
              ├─ child permission context 생성 (부모보다 보수적)
              ├─ child AppState 생성
              ├─ child system prompt 생성
              ├─ child agentLoop 실행
              └─ SubAgentResult 반환
          └─ AgentTool.formatResult()
              └─ parent에 tool_result(summary/finalText/usedTools/usage) 주입
```

### 부모-자식 컨텍스트 규칙
- 부모 메시지 히스토리는 child에게 그대로 넘기지 않는다.
- child는 아래만 입력으로 받는다.
  - child 전용 system prompt
  - 사용자가 위임한 `prompt`
  - 필요한 경우 현재 `cwd`
- child 결과는 부모에 아래만 반환한다.
  - `finalText`
  - `summary`(`finalText`의 deterministic 축약본; 추가 LLM 요약 호출 없음)
  - `usedTools`
  - `usage`
  - `turns`

### 권한/안전성 규칙
- depth 기본값: `1`
- child `maxTurns` 기본값: `6` (상한 `8`)
- child 내부에서 `AgentTool` 사용 금지
- child tool allowlist 기본값:
  - `Glob`
  - `Grep`
  - `FileRead`
  - `MemoryRead`
  - `Bash` (classifier상 allow/read-only 명령만 실질 허용)
- child permission context:
  - 부모 `denyAll` → child `denyAll`
  - 부모 `default` / `acceptAll` → child는 별도 child rules를 거친 보수적 context 사용
  - child에서 `ask`가 필요한 동작은 결과적으로 `deny`되어야 함
- parent abort → child abort 전파

### 최소 계약 변경안
```ts
// src/agent/subagent-types.ts
export interface SubAgentRequest {
  prompt: string;
  allowedTools?: string[];
  maxTurns?: number;
}

export interface SubAgentResult {
  finalText: string;
  summary: string;
  turns: number;
  usedTools: string[];
  usage: Usage;
}

export interface SubAgentRuntime {
  run(request: SubAgentRequest, context: ToolUseContext): Promise<SubAgentResult>;
}
```

```ts
// src/tools/types.ts (MVP draft)
export interface ToolUseContext {
  cwd: string;
  abortSignal: AbortSignal;
  nonInteractive: boolean;
  projectMemory?: ProjectMemoryCore;
  permissionContext?: PermissionCheckContext;
  agentDepth?: number;
  subAgentRuntime?: SubAgentRuntime;
}
```
