# implement_20260419_164624.md

작성 일시: `2026-04-19 16:46:24 KST`

이 문서는 싱글 에이전트를 실제 운영자가 쓰기 편하게 만드는 **Operator UX 1차 보강** 범위를 고정하고, 병렬 구현 시 파일 충돌을 줄이기 위한 작업 문서다.

## 개발 목적
- AgentTool 하위 작업에 parallel-dev 운영 규칙을 자동 주입해 서브 에이전트의 파일 충돌을 줄인다.
- 세션 결과를 Markdown / PR 본문으로 내보내 작업 공유와 리뷰 준비를 쉽게 만든다.
- 장시간 무응답 작업을 감지하는 Watchdog을 추가해 싱글 에이전트 운용 안정성을 높인다.

## 개발 범위
| WS | 항목 | 핵심 산출물 | 1차 소유 경로 |
|---|---|---|---|
| Phase 0 | 공유 계약 고정 | Operator UX 옵션/타입/문서 계약 | `src/agent/*types*`, `dev-plan/*` |
| WS-1 | parallel-dev AgentTool guidance | child agent system prompt에 소유권/비소유권/계약 규칙 주입 | `src/agent/system-prompt.ts`, `src/agent/subagent-types.ts`, `src/tools/agent/agent-tool.ts`, `tests/*agent*` |
| WS-2 | Session export | 세션 Markdown / PR 본문 포맷터 + CLI/TUI 명령 | `src/session/export.ts`, `src/tui/slash-commands.ts`, `src/index.ts`, `tests/session-export.test.ts` |
| WS-3 | Watchdog | idle/progress timeout 감지 + 이벤트/CLI/TUI 연결 | `src/agent/watchdog.ts`, `src/agent/loop.ts`, `src/index.ts`, `src/tui/repl.tsx`, `tests/watchdog.test.ts` |

## 공통 진행 규칙
- Phase 0에서 공유 타입/옵션 이름을 먼저 고정한다.
- WS-1~WS-3은 가능한 병렬로 진행하되, `src/index.ts`, `src/tui/repl.tsx`, `src/tui/slash-commands.ts`는 최종 통합 단계에서 메인 작업자가 병합한다.
- 각 WS는 소유 경로 밖의 파일을 수정하지 않는다. 불가피한 공유 파일 수정은 구현 요약에 명시한다.
- 기본 동작은 기존과 동일해야 한다. 새 기능은 명시 옵션 또는 slash command로만 동작한다.
- 문서에 없는 범위 확장은 하지 않는다.
- 자체 테스트 후 `bun run typecheck`, `bun run build`, `bun test`로 최종 회귀를 확인한다.
