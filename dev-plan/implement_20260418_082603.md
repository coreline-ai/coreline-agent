이 문서는 Proxy 운영 polish 작업의 범위를 고정하고, 현재 구현된 proxy 기능을 실제 운영/연동 관점에서 더 예측 가능하게 다듬기 위한 작업 문서다.

## 개발 목적
이미 구현된 proxy v2 / hosted tools / 라우팅 기능을 “개발자 기능” 수준에서 “운영 가능한 인터페이스” 수준으로 끌어올린다.

## 개발 범위
- proxy auth / logging / request tracing 정리
- batch 제한 정책과 보호 장치 정리
- provider/hosted-tool 지원 여부를 더 명확히 노출
- human input mode 지원 여부를 명시적 계약으로 정리
- curl/example 문서와 운영용 주의사항 반영

## 제외 범위
- proxy 아키텍처 전면 재설계
- MCP와 proxy의 완전 통합 재설계
- 새로운 provider type 추가
- proxy를 full orchestration gateway로 확장
- UI/TUI 기능 추가

## 공통 진행 규칙
- 먼저 운영 계약(auth/logging/limit)을 고정한 뒤 구현한다.
- proxy 응답 shape는 가능한 한 기존 엔드포인트와 일관되게 유지한다.
- human input mode는 구현하더라도 최소 범위로 제한하고, 미구현이면 명시적 not-supported 응답을 준다.
- 체크박스 상태를 실제 진행 상태와 맞게 업데이트한다.
- 문서에 없는 범위 확장은 하지 않는다.
