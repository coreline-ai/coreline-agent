## 개발 목적
P0~P3 범위를 문서/코드 차이 기준으로 재정의하고, 최대한 충돌 없이 병렬 구현 가능한 작업 스트림으로 재구성한다.

## 개발 범위
- P0: 배포/실행 차단 이슈와 문서-코드 불일치 해소
- P1: 런타임 실행 흐름, 비대화형 입력 흐름, 프롬프트 입력 UX 보강

- P3: AGENT.md/CLAUDE.md + 프로젝트별 Memory 시스템 구현
- 병렬 작업을 위한 파일 소유권, 계약 경계, 병합 순서까지 함께 정의

## 제외 범위
- MCP, plugin, voice, remote agent 등 문서 외 신규 기능 추가
- 문서에 없는 대규모 아키텍처 재작성
- 실제 구현 없이 계획 범위를 넘는 상세 설계 문서 확장
- `src/tools/agent/agent-tool.ts` 신규 구현
