# implement_20260424_224740.md

작성 일시: `2026-04-24 22:47:40 KST`

이 문서는 `coreline-cli` 전문가 재리뷰 결과를 바탕으로, `coreline-agent`에 추가 적용할 권장 모듈의 개발 범위를 고정하고 구현이 목적 밖으로 확장되지 않도록 하기 위한 작업 문서다.

## 개발 목적

- `coreline-cli`에서 확인된 추가 적용 후보 중 `coreline-agent`에 실효성이 높고 리스크가 낮은 기능을 V2.1 범위로 이식한다.
- 기존 V1/V2 적용 영역인 Codex auth, Bash security, filesystem hardening, FileRead/FileEdit safety, PostTool hook, model/cost/statusline을 유지하면서 미흡한 안정성 gap을 보완한다.
- 대용량 출력, 메모리 비밀값, MCP resource, 파일 편집 stale-write, 장기 작업 체크리스트, 구조화 사용자 질문 UX를 우선 강화한다.

## 개발 범위

- Secret scanner 고도화
  - `coreline-cli/src/services/teamMemorySync/secretScanner.ts`의 high-confidence secret rule 컨셉을 `src/memory/safety.ts`에 맞게 축소 이식한다.
  - MemoryWrite, auto-summary에 동일 detector를 연결하고 trace/status export 등이 재사용할 수 있는 redaction API를 제공한다.
- 대용량 tool result persistence
  - `coreline-cli/src/utils/toolResultStorage.ts`, `mcpOutputStorage.ts`의 핵심 구조를 `coreline-agent` 세션 디렉터리에 맞춰 구현한다.
  - 큰 Bash/Grep/MCP 결과는 파일 저장 + preview 메시지로 대체한다.
- FileRead/FileEdit 안정성 강화
  - FileRead가 full/partial read state를 기록하고 FileEdit가 read-before-write 및 mtime/content stale-write를 검증한다.
  - 기존 encoding/atomic write/diff/backup 구현과 충돌하지 않게 통합한다.
- MCP resource 도구 추가
  - MCP transport/manager에 `resources/list`, `resources/read`를 추가한다.
  - `ListMcpResources`, `ReadMcpResource` 내장 도구를 추가하고 binary/blob는 파일 저장으로 처리한다.
- TodoWrite 세션 체크리스트 도구 추가
  - root/child context별 간단한 session todo state를 제공한다.
  - all completed 시 clear, 검증 누락 nudge는 최소 규칙으로 시작한다.
- AskUserQuestion 구조화 질문 도구 추가
  - 1~3개 질문, 2~4개 option, 단일 선택 중심으로 시작한다.
  - TUI permission prompt와 별도 이벤트/응답 경로를 구현한다.
- 문서/테스트 갱신
  - README, AGENTS, docs/implementation-status.md, 관련 docs를 실제 구현과 맞춘다.
