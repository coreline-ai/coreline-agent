# implement_20260418_082604.md

작성 일시: `2026-04-18 08:26:04 KST`

이 문서는 AgentTool v3 작업의 범위를 고정하고, 이미 구현된 v2 기능을 실사용 품질과 가시성 중심으로 다듬기 위한 작업 문서다.

## 개발 목적
이미 동작하는 AgentTool v2(병렬 child, provider override, write child 승인, coordinator)를 기반으로, 실제 사용 시 디버깅/결과 전달/승인 UX를 더 이해 가능하게 만든다.

## 개발 범위
- child 실행 기록/디버그 보기 강화
- child 결과 전달 구조(artifact/result handoff) 개선
- write-capable child 승인 UX 강화
- coordinator 결과 품질 및 사용자 표시 개선
- 관련 session/TUI/테스트 보강

## 제외 범위
- depth > 1 재귀 실행
- 완전 자동 멀티 에이전트 스웜
- child transcript 전체 실시간 스트리밍 기본화
- remote/distributed agent runtime
- planner와 AgentTool의 완전 통합 재설계

## 공통 진행 규칙
- v2에서 지킨 안전 원칙(depth 1, parent보다 강한 권한 금지)은 그대로 유지한다.
- v3의 목표는 “더 많은 child”가 아니라 “더 잘 보이고 더 이해되게 만드는 것”이다.
- 디버그 정보는 기본 UX를 깨지 않도록 opt-in 또는 compact 형태로 노출한다.
- 체크박스 상태를 실제 진행 상태와 맞게 업데이트한다.
- 문서에 없는 범위 확장은 하지 않는다.
