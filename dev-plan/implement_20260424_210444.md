# implement_20260424_210444.md

작성 일시: `2026-04-24 21:04:44 KST`

이 문서는 `coreline-agent` V2 후속 개발 범위를 고정하고, 2026-04-24에 완료한 `coreline-cli` 적용 가능 모듈 1~6 구현 이후 작업이 목적 밖으로 확장되지 않도록 하기 위한 작업 문서다.

## 개발 목적

V2의 목적은 새 대형 기능을 무리하게 추가하는 것이 아니라, 이미 적용된 안전성/운영성 모듈을 제품 수준으로 안정화하고, 사용자가 체감할 수 있는 관측성·권한 판단·상태 표시 품질을 높이는 것이다.

핵심 목표는 다음과 같다.

1. Bash security/parser V2로 false positive/false negative를 줄인다.
2. filesystem/FileRead/FileEdit hardening을 symlink·realpath·binary·atomic write 관점으로 보강한다.
3. Hook Engine V2에서 command hook을 바로 공개하지 않고, permission-gated 안전 실행 기반을 먼저 만든다.
4. Codex/model/cost/statusline 정보를 provider/runtime/TUI까지 일관되게 노출한다.
5. V2 smoke matrix와 문서 기준선을 정리해 이후 A2A/MCP/plugin/LSP 같은 큰 기능의 기반을 만든다.

## 개발 범위

- `src/permissions/bash-safety.ts`, `src/permissions/classifier.ts`의 parser/분류 정밀도 개선
- `src/permissions/filesystem-hardening.ts`, `src/permissions/engine.ts`의 realpath/symlink/ancestry hardening 보강
- `src/tools/file-read/*`, `src/tools/file-edit/*`의 binary/atomic/large-file 안정성 보강

- `src/providers/codex-auth.ts`, `src/providers/codex-backend.ts`, provider metadata 흐름에 Codex config/quota/model reasoning 정보 노출
- `src/agent/cost-tracker.ts`, `src/tui/status-bar.tsx`, runtime status snapshot에 model/cost/quota/statusline 개선 반영
- `tests/*`에 V2 회귀 테스트 추가
- `README.md`, `AGENTS.md`, `docs/implementation-status.md`, `docs/hook-engine.md`, OAuth/status 관련 문서 갱신

- `dist/`, `node_modules/`, `.env`는 수정하지 않는다.

## 문서 기반 제약 사항

- TypeScript ESM import에는 `.js` 확장자를 포함한다.
- V2 시작 기준선은 `bun run typecheck`, `bun run build`, `bun test` 기준 `801 pass / 0 fail / 130 files`였고, V2 완료 기준선은 `845 pass / 0 fail / 131 files`다.
- 기존 테스트는 skip/delete하지 않는다.
- Hook failures는 fail-open이어야 하며, explicit `blocking: true`만 실행 전 차단한다.
- command hook은 보안상 별도 permission-gated 경로를 통과해야 하며, ambient credential 전달을 기본 금지한다.
- `PermissionEngine`의 우선순위는 deny > ask > allow를 유지한다.
- `acceptAll` 정책을 변경할 경우 기존 계약 변경으로 보고 별도 테스트와 문서화를 필수로 한다.
- provider/model/cost/statusline 변경은 provider API 호출 흐름을 깨지 않아야 한다.
